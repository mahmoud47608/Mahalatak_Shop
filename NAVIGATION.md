# دليل نظام التنقل في محلاتك

## نظرة عامة

نظام التنقل في التطبيق مبني على 6 ملفات أساسية:

```
navigation/
  Route.kt              ← تعريف كل الشاشات
  AppNavigator.kt       ← إدارة التنقل والتابات
  NavigationHost.kt     ← ربط كل شاشة بالكود بتاعها
  AppBottomBar.kt       ← شريط التنقل السفلي
  graphs/
    MainNavGraph.kt     ← إدارة التابات الخمسة الرئيسية

features/main/
  App.kt                ← نقطة البداية اللي بتجمع كل حاجة
```

---

## الملفات بالتفصيل

---

### 1. Route.kt

الملف ده بيعرّف **كل شاشة** في التطبيق.

كل شاشة ليها نوع واحد من 3:

**شاشات التسجيل والدخول:**

- Splash
- Login
- Register
- Activation (بتاخد رقم الموبايل وفلاجات)
- ForgotPassword
- ResetPassword
- PickLocation

**التابات الخمسة الرئيسية:**

- Home
- Products
- Orders
- Chat
- Account

**شاشات فرعية (بتتفتح فوق التابات):**

- OrderDetail
- ChatDetail
- AddProduct
- Settings
- Profile
- وغيرهم...

الملف كمان فيه خاصيتين مساعدة:

- `isTabRoute` — بترجع `true` لو الشاشة واحدة من التابات الخمسة
- `isAuthScreen` — بترجع `true` لو الشاشة من شاشات التسجيل (عشان نعرض صورة الخلفية)

---

### 2. AppNavigator.kt

ده **المخ** اللي بيتحكم في التنقل. فيه حاجتين أساسيتين:

**الأولى — `backStack` (قائمة الشاشات):**

تخيلها زي كومة ورق. كل شاشة جديدة بتتحط فوق اللي قبلها:

```
[Home]                         ← بداية
[Home, OrderDetail]            ← فتحت تفاصيل طلب
[Home, OrderDetail, ChatDetail] ← فتحت شات
```

لما تضغط رجوع، الشاشة اللي فوق بتتشال:

```
[Home, OrderDetail]            ← رجعت من الشات
[Home]                         ← رجعت من تفاصيل الطلب
```

العمليات المتاحة:

| العملية      | الوظيفة                                  |
|--------------|------------------------------------------|
| `push`       | بتحط شاشة جديدة فوق الكومة               |
| `pop`        | بتشيل الشاشة اللي فوق (رجوع)             |
| `replace`    | بتبدل الشاشة اللي فوق بشاشة تانية        |
| `replaceAll` | بتمسح كل الكومة وتحط شاشة واحدة جديدة    |
| `popUntil`   | بتفضل تشيل شاشات لحد ما توصل لشاشة معينة |

**الثانية — `currentTab` (التاب المختار):**

ده متغير مستقل عن الكومة.
لما بتنتقل بين التابات الخمسة، الكومة مش بتتغير خالص.
بس `currentTab` بيتغير، والشاشة بتتبدل.

**ليه كده؟**

لأن الكومة مربوطة بحاجة اسمها `NavDisplay`.
لو غيّرنا الكومة كل مرة ننتقل بين التابات،
`NavDisplay` هيفتكر إن الشاشة اتغيرت وهيعمل شغل كتير = لاج.

لما نغيّر `currentTab` بس من غير ما نلمس الكومة،
`NavDisplay` مش بيحس بأي حاجة = سرعة.

---

### 3. NavigationHost.kt

الملف ده بيوصل كل شاشة بالكود بتاعها.

بيستخدم `NavDisplay` اللي بيقرأ الكومة وبيعرض الشاشة اللي فوق.

إعدادات مهمة:

- **الأنيميشن موقوفة** — بنقول لـ `NavDisplay` ميعملش أي انتقال متحرك
  عشان ده كان سبب رئيسي في اللاج

- **حفظ الحالة** — `NavDisplay` بيحافظ على بيانات كل شاشة في الكومة.
  يعني لو فتحت شاشة التسجيل وكتبت بيانات وبعدين روحت لشاشة تانية
  ورجعت، هتلاقي البيانات موجودة.

الشاشات متقسمة في مجموعات عشان الكود يبقى منظم:

| المجموعة | الشاشات                              |
|----------|--------------------------------------|
| Auth     | التسجيل والدخول ونسيت الباسورد       |
| Detail   | تفاصيل الطلب والشات وإضافة منتج      |
| Profile  | البروفايل والتعديل                   |
| Settings | الإعدادات وتغيير الرقم والباسورد     |
| More     | الموظفين والشكاوى والعروض والكوبونات |
| Tabs     | التابات الخمسة الرئيسية              |

### 4. MainNavGraph.kt

الملف ده بيدير **التابات الخمسة الرئيسية** بشكل خاص.

**المشكلة اللي بيحلها:**

لو كل مرة تنتقل لتاب الشاشة تتبني من الصفر،
هتفقد مكان السكرول والبيانات المحمّلة وكل حاجة.
ده كمان بيعمل لاج.

**الحل:**

كل التابات الخمسة موجودين في نفس الوقت،
بس التاب المختار ظاهر والباقي مخفيين.

تخيلها زي 5 ورقات متراصة فوق بعض:

```
┌─────────────┐
│   Orders    │  ← ظاهر (zIndex = 1, alpha = 1)
├─────────────┤
│    Home     │  ← مخفي (zIndex = 0, alpha = 0)
├─────────────┤
│  Products   │  ← مخفي
├─────────────┤
│    Chat     │  ← مخفي
├─────────────┤
│   Account   │  ← مخفي
└─────────────┘
```

**تفاصيل مهمة:**

- `alpha = 0` — بيخفي الشاشة بصرياً (مش شايفها)
- `zIndex = 1` — بيخلي الشاشة الظاهرة فوق الكل (بتستقبل الضغطات)
- `zIndex = 0` — الشاشات المخفية تحت (مش بتستقبل أي ضغطات)

**بدون `zIndex` كان بيحصل مشكلة:**

الشاشة المخفية كانت بتستقبل الضغطات
لأن `alpha = 0` بيخفي بصرياً بس مبيوقفش الضغطات.

**التحميل الكسول:**

مش كل التابات بتتحمل مرة واحدة.
كل تاب بيتحمل **أول مرة بس** تزوره.
بعد كده بيفضل حي في الذاكرة.

### 5. AppBottomBar.kt

شريط التنقل السفلي بالدائرة المتحركة.

**المكونات:**

- 5 أزرار: الرئيسية، الطلبات، المنتجات، الشات، المزيد
- دائرة ملونة بتتحرك فوق الزر المختار
- قطع (cutout) في الشريط مكان الدائرة

**الأنيميشن:**

- الدائرة بتتحرك بأنيميشن spring (زنبرك — بتهتز شوية وبعدين تقف)
- القطع في الشريط بيتحرك مع الدائرة
- الأيقونة جوه الدائرة بتتغير بأنيميشن crossfade

**حاجة مهمة:**

الشريط بيقرأ `currentTab` مش `currentRoute`.
كده لما شاشة فرعية تتفتح (زي تفاصيل طلب)
الشريط بيختفي بس التاب المختار مبيتغيرش.

---

### 6. App.kt

نقطة البداية اللي بتجمع كل حاجة.

**الترتيب:**

1. بتعمل `AppNavigator` وبتحفظه
2. بتوفره لكل الشاشات عن طريق `LocalNavigator`
3. بتعرض `Scaffold` فيه:
    - شريط التنقل السفلي (بيظهر بس في التابات الرئيسية)
    - صورة الخلفية (بتظهر بس في شاشات التسجيل)
    - `NavigationHost` (بيعرض الشاشة الحالية)
    - شاشة التحميل (لو فيه عملية جارية)

## رحلة التنقل — مثال عملي

### أول ما التطبيق يفتح:

```
الكومة: [Splash]
```

الاسبلاش بتشيك لو مسجل دخول.
لو أيوه بتعمل `replaceAll(Home)`:

```
الكومة: [Home]
currentTab: Home
```

### التنقل بين التابات:

بتدوس على "الطلبات":

```
الكومة: [Home]          ← مش اتغيرت!
currentTab: Orders       ← ده اللي اتغير
```

التطبيق بيعرض شاشة الطلبات وبيخفي الرئيسية.

بتدوس على "المنتجات":

```
الكومة: [Home]          ← لسه مش اتغيرت
currentTab: Products     ← اتغير تاني
```

### فتح شاشة فرعية:

من المنتجات بتدوس "إضافة منتج":

```
الكومة: [Home, AddProduct]    ← اتضاف فوق
currentTab: Products            ← مش اتغير
```

الشريط السفلي بيختفي لأن `AddProduct` مش تاب.

### الرجوع:

بتدوس رجوع:

```
الكومة: [Home]          ← AddProduct اتشال
currentTab: Products     ← لسه Products
```

الشريط السفلي بيرجع يظهر وبتلاقي المنتجات زي ما سبتها.

## ملخص القرارات المهمة

**1. ليه الأنيميشن متوقفة في NavDisplay؟**

لأن `NavDisplay` بيعمل أنيميشن بين الشاشات تلقائياً.
ده كان بيحمّل شاشتين في نفس الوقت (القديمة والجديدة) = لاج.

**2. ليه `switchTab` مش بيغيّر الكومة؟**

لأن أي تغيير في الكومة بيخلي `NavDisplay` يعمل شغل.
لما نغيّر `currentTab` بس، `NavDisplay` مش بيحس بأي حاجة = سرعة.

**3. ليه التابات مخفية بـ alpha مش متشالة من الكود؟**

لو شيلناها من الكود (زي ما نعمل if/else)
هتضيع كل البيانات والسكرول والحالة.
الـ alpha بيخفيها بصرياً بس وبتفضل حية في الذاكرة.

**4. ليه محتاجين zIndex؟**

لأن `alpha = 0` بيخفي الشاشة بصرياً بس مبيوقفش الضغطات.
الـ `zIndex` بيخلي الشاشة الظاهرة فوق الكل فهي اللي بتاخد الضغطات.
